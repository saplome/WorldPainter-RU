# WorldPainter RU — финальный проход локализации v3

## Что исправлено в этом проходе

- Исправлен краш `NullPointerException` в `App.setTextIfDifferent(...)`, который возникал при движении мыши по карте, если `JLabel.getText()` возвращал `null`.
- Убраны опасные `setText(null)` в основных Swing-компонентах. Вместо `null` теперь используется пустая строка, чтобы интерфейс не падал на сравнении текста.
- Добавлена null-safe проверка в `CustomMaterialDialog` через `java.util.Objects.equals(...)`.
- Дорусифицированы строки статус-бара:
  - `Slope: ...` -> `Уклон: ...`
  - `Auto biome: ...` -> `Автобиом: ...`
  - `Border` -> `Граница`
- Дорусифицировано отображение измерения `End` в диалогах экспорта/тестового экспорта.
- Дорусифицировано значение `none` в списке смешанных материалов.
- Проверено соответствие ключей `strings.properties` и `strings_ru.properties`: отсутствующих ключей в русском файле не найдено.

## Что осталось проверить вручную в GUI

- Все панели инструментов и их настройки, особенно редкие операции.
- Диалоги экспорта/импорта.
- Панели пользовательских слоёв, растений, объектов BO2/BO3/NBT/Schem.
- Системные файловые диалоги Windows: часть текста может приходить от ОС/LookAndFeel и не всегда управляется проектом.
- Сообщение JIDE про Unauthorized usage не является проблемой русификации и не исправлялось.

## Проверка после распаковки

Из корня проекта:

```cmd
mvn clean package
```

Запуск GUI:

```cmd
mvn -pl WPGUI exec:exec
```

Если снова появится ошибка, нужно прислать полный stack trace и указать действие, после которого она появилась.
